# Test Design Review Checklist

## Context

- [ ] Test design input is identified.
- [ ] Defect pattern catalog availability is documented.
- [ ] Regression viewpoint catalog availability is documented.
- [ ] Missing inputs are listed.

## Coverage Mapping

- [ ] Existing test areas are summarized.
- [ ] Viewpoint coverage is marked covered/partial/missing/unknown.
- [ ] Unknown is used when evidence is unavailable.
- [ ] Missing coverage names the exact missing condition.

## Findings

- [ ] Findings are ordered by severity.
- [ ] Each finding has a related viewpoint or defect pattern.
- [ ] Each finding has evidence type.
- [ ] Each finding has a concrete required change.
- [ ] No finding says only "add more tests".

## Output

- [ ] Review summary is actionable.
- [ ] Coverage matrix is included.
- [ ] Open questions are included.
- [ ] Recommended test additions are included.
