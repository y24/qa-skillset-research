# Step 02: Classify Defects

## Goal

Classify each defect into consistent QA categories.

## Instructions

1. Use `defect-taxonomy.md` as the first taxonomy.
2. Assign each defect:
   - functional area
   - defect type
   - trigger condition
   - quality characteristic
   - detection phase
   - severity
   - likely test gap, if supported
3. Preserve original wording for important observations.
4. If a classification is uncertain, mark it as `unknown` or `hypothesis`.

## Output Fragment

Create a classification summary:

| Category | Count | Representative Defects | Notes |
|---|---:|---|---|

Also capture any records that cannot be classified and why.
