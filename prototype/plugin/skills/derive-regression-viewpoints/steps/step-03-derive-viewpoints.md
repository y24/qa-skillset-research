# Step 03: Derive Viewpoints

## Goal

Turn each prioritized pattern into concrete regression viewpoints.

## Instructions

For each selected pattern, create one or more viewpoints with:

- stable viewpoint ID
- title
- priority
- source pattern IDs
- source defect IDs
- applicable areas
- test conditions
- suggested test viewpoints
- evidence and rationale
- recommended cadence

## Viewpoint Quality Criteria

A viewpoint is good when it:

- names a failure condition, not just a module
- contains variable conditions to test
- has a clear oracle or expected outcome
- can be reused in future test design reviews
- cites pattern/defect evidence

## Output Fragment

Draft viewpoint sections for `regression-viewpoint-catalog.md`.
